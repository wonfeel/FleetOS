FleetOS - быстрый старт

4 шага, чтобы поднять мост и первый компьютер. Подробности - полный гайд.

1. Запустите мост на Windows-ПК

Двойной клик:

windows\start_bridge.bat

(игра на другом ПК/по Radmin VPN - вместо этого windows\start_bridge_mc.bat).

Откройте http://127.0.0.1:8787/ - это дашборд.

2. Установите на компьютер в игре

wget http://<адрес-моста>:8787/install.lua install.lua
install http://<адрес-моста>:8787

Адрес можно не указывать, если мост на 127.0.0.1:8787. Всё остальное - код, config.lua, автозапуск - компьютер настроит сам.

Разрешите адрес моста в CC:Tweaked, если ещё не делали: computercraft-server.toml -> [http.rules]. Иначе будет «HTTP API is disabled».

3. Готово

Через несколько секунд узел появится в списке «Fleet» на дашборде - оттуда запуск/деплой приложений, терминал, переименование.

Адрес/ключ моста можно поменять и позже, без переустановки - наберите в консоли компьютера (или через панель Terminal) bridge http://новый-адрес:8787 ключ. Команда сама перезапустит fleetbridge с новыми настройками; bridge без аргументов покажет текущие, bridge clear вернёт к тем, что в config.lua. Для эмуляции на ПК (без Minecraft) то же самое можно передать прямо при запуске: run.bat http://127.0.0.1:8787 ключ.

4. Защита (по желанию)

По умолчанию мост без пароля - годится только для 127.0.0.1 или доверенного VPN (Radmin). Если мост доступен кому-то ещё, включите ключ:

  1. При запуске моста: set FLEET_BRIDGE_KEY=мой-ключ перед py windows\bridge_server.py 8787 (или раскомментируйте строку в start_bridge_mc.bat).
  2. При установке компьютера - вторым аргументом: install http://<мост>:8787 мой-ключ.
  3. В дашборде - поле «API key» рядом с адресом моста.
Без ключа без разницы, кто угодно в сети моста сможет выполнять код на ваших игровых компьютерах - не выставляйте мост дальше localhost/VPN, если ключ не включён.
