FleetOS - руководство

Мини-ОС для компьютеров CC:Tweaked (Minecraft) + мост/дашборд для управления с Windows-ПК.
Обзор 1. Мост (bridge_server.py) 2. Установка на компьютер 3. Дашборд 4. Защита (API-ключ) 5. fleetctl.py 6. Raytower 7. Проблемы

Обзор

FleetOS превращает набор компьютеров CC:Tweaked в управляемый «флот»: у каждого свой мини-ядро (fleetos.lua), которое запускает приложения (apps/) и умеет опрашивать ваш Windows-ПК на предмет команд.

Все компьютеры равноправны - master/slave нет. Каждый узел напрямую общается с мостом по HTTP через свой уникальный id (из config.lua). Мост (bridge_server.py) просто хранит очередь команд на каждый узел и последний полученный от него статус.

ЧастьЧто это
game/fleetos.luaЯдро: кооперативная многозадачность, менеджер приложений, зеркалирование терминала на монитор с сенсорными кнопками (запуск/свернуть/восстановить/закрыть у каждого приложения).
game/install.luaЗагрузчик для нового компьютера - единственный файл, который качается вручную.

Есть отдельный гайд именно по триангуляции вышек (Raytower) - файл guide.html рядом с этим. Здесь - про флот целиком, там - только про raytower.lua.
Админ- и security-функции (ключ только для чтения, /health//metrics, экспорт/импорт флота, откат ядра, подпись Raytower и т.д.) - отдельный гайд hardening_guide.html.

1. Запуск моста

Нужен установленный Python. Запускайте одним из двух вариантов:

Только локально (тест/один ПК с игрой)

windows\start_bridge.bat

Слушает 127.0.0.1:8787. Подходит, если Minecraft и мост работают на одной машине.

Через Radmin VPN (игра на другом ПК/с друзьями)

windows\start_bridge_mc.bat

Слушает на вашем Radmin-адресе, чтобы реальный компьютер в игре мог достучаться через VPN. В game/config.lua каждого узла поле bridgeUrl должно указывать на этот же адрес.

Важно: CC:Tweaked по умолчанию блокирует http-запросы к локальным/LAN-адресам. Нужно разрешить адрес вашего моста в computercraft-server.toml, секция [http.rules] - иначе fleetbridge.lua будет писать «HTTP API is disabled» или получать отказ.

После запуска откройте http://127.0.0.1:8787/ в браузере - это и есть дашборд.

2. Установка на новый компьютер (в игре)

На свежем компьютере CC:Tweaked, у которого ещё ничего нет:

wget http://<адрес-моста>:8787/install.lua install.lua
install http://<адрес-моста>:8787

Адрес моста можно не указывать - тогда по умолчанию берётся http://127.0.0.1:8787. install сам:

  • скачивает fleetos.lua и базовые приложения (shell, fleetbridge);
  • создаёт стартовый config.lua с уникальным id (если его ещё нет - повторный запуск ничего не затирает);
  • прописывает startup.lua, чтобы после перезагрузки FleetOS запускался сам;
  • сразу запускает ядро.

Если на мосту включена защита ключом (раздел 4 ниже), передайте его вторым аргументом:

install http://<адрес-моста>:8787 <api-key>

После установки id/role/список автозапуска можно менять прямо через панель «Remote files» дашборда (путь config.lua) или переименовать узел одной кнопкой (раздел 3).

Адрес/ключ моста можно поменять и позже, без переустановки - команда bridge <url> [key] в консоли компьютера (или через панель Terminal на дашборде). Она пишет отдельный файл bridge_override.txt (побеждает config.lua) и сама перезапускает fleetbridge с новыми настройками. bridge без аргументов - показать текущий; bridge clear - вернуться к значениям из config.lua.

3. Дашборд

Всё управление - с одной страницы http://<адрес-моста>:8787/.

Fleet - выбор компьютера

Список появляется сам, как только узел хотя бы раз опросил мост. Клик по строке компьютера выбирает его - панели «Terminal» и «Explorer» ниже работают именно с ним.

  • Update fleetos - перекачивает свежий fleetos.lua и перезагружает узел (ядро не «приложение», обычный деплой его не трогает).
  • Rename - меняет id узла (пишется в отдельный файл node_id.txt, не трогая ваш config.lua) и перезагружает его.

Папки: в столбце «Folder»/«Папка» можно распределить компьютеры по своим папкам (создаются на лету через «+ new folder...»). Заголовок каждой папки сворачивается кликом и даёт групповые действия - обновить fleetos, переименовать или разгруппировать сразу все компьютеры внутри. У каждой папки свой цвет - граница слева и подсветка заголовка помогают отличать их на глаз.

Массовые действия: отметьте галочками несколько компьютеров (в любых папках, не обязательно в одной) - появится панель с перемещением сразу всех отмеченных в папку, общим «Update fleetos», а также запуском/остановкой приложения по имени сразу на всех отмеченных.

Позиция: если у узла есть доступ к GPS-станциям (gps.locate() - нужна сеть из 4+ компьютеров с беспроводными модемами, см. документацию CC:Tweaked по GPS), столбец «Position»/«Позиция» покажет его координаты. Без GPS-станций там просто будет «неизвестно» - остальной функционал дашборда от этого не зависит.

Terminal

Показывает live-вывод выбранного компьютера (то, что он печатает), и позволяет ввести команду - как будто вы печатаете прямо в его консоли. Команды ядра: list/status, run <app>, kill <app>, minimize <app>, restore <app>, apps - плюс любая настоящая CraftOS-программа.

Explorer

Проводник в стиле Windows: переход по папкам (хлебные крошки сверху), создание папок/файлов, открытие файла на редактирование прямо в браузере, сохранение и удаление (папка удаляется вместе со всем содержимым - дашборд переспросит перед этим). Работает с любым файлом/папкой, не только с заранее известными путями.

Выпадающий список «Browse»/«Источник» переключает, ЧТО именно вы просматриваете: выбранный компьютер в игре, либо сам ваш ПК - папку game/ (шаблоны, из которых деплоятся приложения) или windows/compute/ (Python/C++ скрипты). Просмотр «этого ПК» не ждёт опроса узла - открывается мгновенно.

  • Переименование - маленький значок ✏️ рядом с именем превращает его в поле ввода; Enter или клик в сторону подтверждает, Escape отменяет.
  • Запуск (Run) - только для файлов на живом компьютере: запускает файл так, будто его путь ввели прямо в консоли.
  • Скачивание (Download) - файл скачивается как есть, папка - упаковывается в .zip прямо в браузере и скачивается одним архивом.
  • Открытие файла приложения (при просмотре game/apps/... на ПК) - под редактором появляются кнопки Run/Kill/Deploy с выбором целевого компьютера, чтобы не переключаться в отдельную таблицу.

Перетаскивание (drag-and-drop): перетащите файл или целую папку прямо на область проводника - она загрузится (рекурсивно, с созданием вложенных папок) туда, где вы сейчас находитесь, будь то компьютер в игре или папка на ПК.

Язык интерфейса - кнопки EN/RU в правом верхнем углу дашборда переключают весь текст страницы; выбор запоминается в браузере.

Create a new app

Пишете код прямо в браузере - создаётся новый файл в game/apps/ на вашем ПК (группа common/raytower/«other» - просто для порядка в списке). Дальше откройте его в Explorer («Этот ПК - game/»), чтобы редактировать/деплоить - отдельной таблицы для этого больше нет.

Macros

Один клик = несколько команд подряд (например «Redeploy fleetbridge»).

Монитор в игре

Если к компьютеру подключён продвинутый монитор (Advanced Monitor) - на нём само появляется панель задач с кнопками в стиле окон Windows:

СостояниеКнопкиЧто означает
остановлено[>]запустить
запущено[_] [X]свернуть / закрыть
свёрнуто[^] [X]восстановить / закрыть

«Свернуть» - это просто визуальный режим (как в Windows): приложение как работало, так и работает, скрывается только надпись. Тап по заголовку сворачивает/разворачивает весь список - вместо него показывается лента лога на весь монитор.

4. Защита по API-ключу

По умолчанию защиты нет вообще - мост рассчитан на 127.0.0.1 или доверенный VPN (Radmin). Если хотите обезопасить доступ (например, играете через Radmin с другими людьми), включите ключ:

  1. Задайте переменную окружения перед запуском моста: 
    set FLEET_BRIDGE_KEY=мой-секретный-ключ
py windows\bridge_server.py 8787
    (в start_bridge_mc.bat есть готовая закомментированная строка - раскомментируйте и впишите свой ключ).
  2. Впишите тот же ключ в config.lua каждого узла: 
    apiKey = "мой-секретный-ключ",
  3. В дашборде - поле «API key» рядом с адресом моста (раздел «Bridge address»), сохраняется в браузере.
  4. При установке нового компьютера - вторым аргументом команды install (раздел 2).
  5. Для fleetctl.py - та же переменная окружения FLEET_BRIDGE_KEY.
Без ключа всё продолжает работать как раньше - это опция, а не обязательное требование, пока адрес - 127.0.0.1.
Ключ можно не придумывать самому. Если запустить мост на адресе, отличном от 127.0.0.1 (Radmin/LAN через FLEET_BRIDGE_HOST), и FLEET_BRIDGE_KEY не задан - мост сам сгенерирует ключ, один раз сохранит его в windows/bridge_key.txt (переиспользуется при перезапусках) и выведет прямо в консоль при старте. Достаточно скопировать его оттуда - придумывать и вписывать вручную не нужно.
Без ключа не выставляйте мост дальше localhost/VPN. Кто угодно, кто достучится до адреса моста, сможет выполнять код и читать/писать файлы на любом вашем игровом компьютере. Именно поэтому при адресе, отличном от 127.0.0.1, мост либо генерирует ключ сам (см. выше), либо (только если явно указать FLEET_BRIDGE_HOST_ALLOW_INSECURE=1) продолжает без ключа с явным предупреждением.

5. fleetctl.py - консольная альтернатива

Тот же API, что у дашборда, но из командной строки:

python windows\fleetctl.py ping
python windows\fleetctl.py status
python windows\fleetctl.py run <узел|*> <приложение>
python windows\fleetctl.py kill <узел|*> <приложение>
python windows\fleetctl.py deploy <узел|*> <приложение> <url-с-кодом>
python windows\fleetctl.py rollback <узел|*> <приложение>
python windows\fleetctl.py type <узел|*> <текст>
python windows\fleetctl.py shell <узел>

Адрес моста и ключ - через переменные окружения FLEET_BRIDGE_URL / FLEET_BRIDGE_KEY, если не стандартные.

При эмуляции на ПК (без Minecraft) предпочитайте fleetctl.py shell локальному shell внутри узла. Локальная консоль эмуляции (windows\craftos_shim.lua) читает ввод блокирующе - пока в ней ждут ввод «shell>», fleetbridge и все остальные задачи на этом же узле замирают. fleetctl.py shell <узел> - тот же ввод команд, но отдельным процессом через HTTP (как панель Terminal на дашборде), поэтому ничего на узле не блокирует. В реальном Minecraft этой проблемы нет вовсе.

6. Raytower (триангуляция) - отдельная фича

Не связана с управлением флотом - это независимый инструмент через rednet (нужен модем) для определения позиции по направлению лучей от нескольких «вышек». Подробный отдельный гайд - guide.html рядом с этим файлом. Коротко:

  • raytower_master (приложение под FleetOS) - на одном компьютере, считает позицию и умеет рисовать её на своём мониторе.
  • raytower_slave (приложение под FleetOS) - на каждой вышке, отчитывается мастеру.
  • game/raytower.lua - тот же функционал одним файлом, без FleetOS, с командой калибровки (master calibrate) для подбора raytowerForward/raytowerQSign.

7. Частые проблемы

СимптомПричина / решение
«HTTP API is disabled»На этом компьютере вообще не включён http в настройках сервера/CC:Tweaked.
Узел не появляется в дашбордеПроверьте computercraft-server.toml -> [http.rules] - адрес моста должен быть разрешён. Также сверьте bridgeUrl в config.lua с реальным адресом/портом моста.
401 / «unauthorized»На мосту включён FLEET_BRIDGE_KEY, а у узла/дашборда либо не задан, либо не совпадает ключ.
После «Update fleetos» узел долго не появляетсяОбычно перезагрузка занимает всего несколько секунд. Если долго - убедитесь, что мир/сервер сейчас реально загружен и тикает (если сервер выключен, компьютер просто ничего не может сделать, пока вы не зайдёте обратно).
Деплой raytower_master падает «File not found»Ему нужен отдельный файл triangulation.lua, не входящий в обычные приложения - дашборд сам подкладывает его при деплое, ничего делать вручную не нужно.
